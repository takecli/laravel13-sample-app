# 共通コーディング規約

全言語・全作業に適用する。言語別の詳細は `php.md`（`*.php`）/ `typescript.md`（`*.ts` `*.tsx`）を参照。
アーキテクチャ全体像は ルートの `CLAUDE.md` を参照。

## 大原則

- **既存コードの流儀に合わせる。** 命名・レイヤー構成・コメント密度・フォーマットは周囲のファイルに倣う。独自流儀を持ち込まない。
- **最小差分。** 要求された変更だけを行う。ついでのリファクタや無断の広域リネームをしない。
- **推測でなく確認。** 仕様・型・既存実装を読んでから書く。`Team` 機能が全レイヤーのリファレンス実装。
- 応答は**日本語**。

## 実装ワークフロー

1. **調べる** — 類似実装（特に `Team` の縦切り）と既存パターンを把握する。
2. **書く** — パターンを踏襲して実装する。型を必ず付ける。
3. **テストする** — 振る舞いを検証するテストを書く／更新する。
4. **整形する** — フォーマッタを通す（PHP: `task php:pint`）。
5. **裏取りする** — テスト・型チェック・ビルドの**コマンド出力を確認**してから完了とする。

## 完了の定義（Definition of Done）

以下を**コマンド出力で確認**できて初めて「完了」と言う。憶測で「直りました」と言わない。

- [ ] テストが緑（`task php:coverage-text`）
- [ ] **行カバレッジ 100% を維持**
- [ ] フォーマッタ通過（`task php:pint`）
- [ ] フロントは型エラーなし（`npx tsc --noEmit`）／ビルド可（`npm run build`）

## 禁止事項

- デバッグ出力を残す（`dd()` / `var_dump()` / `dump()` / `console.log` / `print_r`）。
- 秘密情報（鍵・トークン・パスワード）をコード／コミットに含める。設定は `.env`。
- 既存の公開シグネチャ・命名・綴り（既存のタイポ含む）を無断で変更する。影響範囲を確認し、必要なら相談する。
- 未使用のコード・import・依存を追加する。
- テストを通すためにアサーションを甘くする／本質でない条件分岐を増やす。

## コメント

- **「なぜ」**を書く（「何を」はコードで表現する）。日本語可。
- 既存ファイルのコメント密度・スタイルに合わせる（本リポは日本語の docblock が標準）。

## セキュリティ

- 外部入力は必ず検証する（PHP: FormRequest、TS: 型＋必要ならランタイム検証）。
- SQL は Eloquent / バインドパラメータ経由。文字列連結でクエリを組まない。
- 認可・認証境界（`api/*`、Keycloak セッション）を意識する。

## Git

- ユーザーの明示指示なくコミット／プッシュしない。
- コミットメッセージは日本語、変更意図が分かる粒度で。
